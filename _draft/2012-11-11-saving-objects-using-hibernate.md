<div dir="ltr" style="text-align: left;" trbidi="on"><h3>Introduction to Hibernate</h3><ol><li>An ORM tool.</li><li>Used in the data layer of application.</li><li>Implement JPA.</li><li>JPA (Java persistence api):- this is set of standards for and java persistence api.</li></ol><h3>The problem with JDBC</h3><ol><li>Mapping member variables to the column.</li><li>Mapping relationships.</li><li>Handling data types. <br /> Example Boolean most of the databases do not have Boolean datatype.   </li><li>Managing the state of object.</li></ol><h3>Saving with an object with hibernate.</h3><ol><li>JDBC database configuration <b>&gt;&gt; Hibernate configuration .cfg.xml file.</b></li><li>The model object <b>&gt;&gt; Annotations or Hibernate mapping file .hbm.xml files.</b></li><li>Service method to create the model object <b>&gt;&gt; Use the hibernate.</b></li><li>Database design. <b>&gt;&gt; No needed.</b></li><li>DAO method to save the object using sql queries <b>&gt;&gt; No needed.</b></li></ol><br /><a name='more'></a>In our first hibernate example .I will save a simple object in database using hibernate. I will explore this tutorial further. <br /><br /><h3>Start Coding</h3><h4>Project Structure</h4><div style="clear: both; text-align: center;"><a href="http://4.bp.blogspot.com/-H-jPpThjrQ4/UKdUgKYBWKI/AAAAAAAAAR4/ulGRYcaV-kg/s1600/project_structure.bmp" imageanchor="1" style="clear: left; float: left; margin-bottom: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-H-jPpThjrQ4/UKdUgKYBWKI/AAAAAAAAAR4/ulGRYcaV-kg/s1600/project_structure.bmp" /></a></div><div style="clear: both; text-align: center;"></div><br /><h4>Book.java</h4>Now, i am going to create a simple POJO class <b>Book.java</b> with some properties and their setters and getters.<br /><pre class="brush: java">  <br />package com.bloggers.model;<br /><br />import java.util.Date;<br /><br />public class Book<br />{<br /> /* for primary key in database table */<br /> private int bookId;<br /> private String bookName;<br /> private Date date;<br /><br /> public int getBookId()<br /> {<br />  return bookId;<br /> }<br /><br /> public void setBookId(int bookId)<br /> {<br />  this.bookId = bookId;<br /> }<br /><br /> public String getBookName()<br /> {<br />  return bookName;<br /> }<br /><br /> public void setBookName(String bookName)<br /> {<br />  this.bookName = bookName;<br /> }<br /><br /> public Date getDate()<br /> {<br />  return date;<br /> }<br /><br /> public void setDate(Date date)<br /> {<br />  this.date = date;<br /> }<br /><br /> <br />}<br /></pre><br />Few point while creating a model object or JavaBeans (Book.java) class.<br /><ul><li>bookId property holds a <b>unique identifier</b> value for a book object. Provides an identifier property if you want to use the full feature set of Hibernate   </li><li>Application need to distinguish objects by identifier.</li><li>Hibernate can access public, private, and protected accessor methods, as well as public, private and protected fields directly.</li><li>no-argument constructor is a requirement for all persistent classes. Hibernate create objects using Java Reflection.</li></ul><!-- =============mapping file ================= --> <br /><h4>Mapping file :-hbm file (Hibernate Mapping file)</h4>Now second step is we have to create a hbm file with extension <b>.hbm.xml</b>. HBM file means <b>Hibernate Mapping File</b>. Mapping file gives all the information to hibernate about table columns etc which are mapped to object. <br /><b>All persistent entity classes need a mapping to a table in the SQL database.</b> <br /><b>Basic structor of hbm file  </b> <br /><pre class="brush: xml"> &lt;hibernate -mapping="-mapping"&gt;<br />   [...]<br />   &lt;/hibernate&gt;<br />  </pre><br /><h4>Note:</h4><ul><li>Hibernate not loads the DTD file from the web, it first look it up from the classpath of the application.</li><li>DTD file is included in hibernate-core.jar</li></ul><br /><br /><h4>Mapping file of Book.java class(Book.hbm.xml)</h4>I will complete this step by step and try to explain all the elements.<br /><ol><li>Add a class element between two hibernate-mapping tags.</li><pre class="brush: xml">  &lt;?xml version="1.0"?&gt;<br />     &lt;!DOCTYPE hibernate-mapping PUBLIC<br />      "-//Hibernate/Hibernate Mapping DTD 3.0//EN"<br />      "http://hibernate.sourceforge.net/hibernate-mapping-3.0.dtd"&gt;<br />  <br />    &lt;hibernate-mapping&gt;<br />     &lt;class name="com.bloggers.model.Book" &gt;<br />      <br />     &lt;/class&gt;<br />    &lt;/hibernate-mapping&gt; <br />    </pre><b>class</b>="com.bloggers.model.Book" .Specify the qualified class name in name attribute of class element.<br /><li>Next, we need to tell Hibernate about the remaining entity class properties. By default, no properties of the class are considered persistent.</li><pre class="brush: xml">  &lt;?xml version="1.0"?&gt;<br />     &lt;!DOCTYPE hibernate-mapping PUBLIC<br />       "-//Hibernate/Hibernate Mapping DTD 3.0//EN"<br />       "http://hibernate.sourceforge.net/hibernate-mapping-3.0.dtd"&gt;<br />   <br />     &lt;hibernate-mapping&gt;<br />      &lt;class name="com.bloggers.model.Book" &gt;<br />       &lt;id name="bookId"/&gt;<br />       &lt;property name="date" column="PUBLISH_DATE"/&gt;<br />      &lt;/class&gt;<br />     &lt;/hibernate-mapping&gt;<br />    </pre><b>name</b>="bookId" attribute of the property element tells Hibernate which getter and setter methods to use in our case hibernate will search for getBookId() , setBookId() etc methods.   </ol><br /><b>Question :</b> Why does the date property mapping include the column attribute, but the bookID does not? <br /><b>Answer :</b> Without the column attribute, Hibernate by default uses the property name as the column name. This works for bookId, however, date is a reserved keyword in most databases so you will need to map it to a different name.<br /><!-- =============configuration file ================= -->  <br /><h4>Create a Hibernate Configuration File (hibernate.cfg.xml)</h4>Hibernate required a configuration file for making the connection with database by default the file name is <b>hibernate.cfg.xml</b><br />We can do hibernate configuration in <b>3</b> different ways and these are :-<br /><ol><li>we can use a simple <b>hibernate.properties</b> file   </li><li>a more sophisticated <b>hibernate.cfg.xml</b> file   </li><li>or even complete programmatic setup</li></ol>I am going to use the 2 option means by using <b>hibernate.cfg.xml</b> file. <br /><pre class="brush: xml"> &lt;?xml version='1.0' encoding='utf-8'?&gt;<br /> &lt;!DOCTYPE hibernate-configuration PUBLIC<br />   "-//Hibernate/Hibernate Configuration DTD 3.0//EN"<br /> --&gt;<br />   &lt;property name="connection.driver_class"&gt;com.mysql.jdbc.Driver&lt;/property&gt;<br />   &lt;property />   &lt;!-- SQL dialect --&gt;<br />   &lt;property name="dialect"&gt;org.hibernate.dialect.MySQL5Dialect&lt;/property&gt;<br /><br />   &lt;!-- Disable the second-level cache  --&gt;<br /> factory responsible for a particular database. If you have several databases, for easier startup you should use several &lt;session-factory&gt; configurations in several configuration files.</li><li>The first four property elements contain the necessary configuration for the JDBC connection.</li><li>The dialect property element specifies the particular SQL variant Hibernate generates.</li><li>In most cases, Hibernate is able to properly determine which dialect to use.</li><li>The hbm2ddl.auto option turns on automatic generation of database schema directly into the database. This can also be turned off by removing the configuration option, or redirected to a file with the help of the SchemaExport Ant task.</li><li>Finally, add the mapping file(s) for persistent classes to the configuration.</li></ol><br /><h4>HibernateUtil.java</h4>Utility class for getting the <b>sessionFactory.</b><br /><pre class="brush: java">  package com.bloggers.util;<br /><br />  import org.hibernate.SessionFactory;<br />  import org.hibernate.cfg.Configuration;<br />  <br />  public class HibernateUtil<br />  {<br />   private static final SessionFactory sessionFactory = buildSessionFactory();<br />  <br />   private static SessionFactory buildSessionFactory()<br />   {<br />    try<br />    {<br />     // Create the SessionFactory from hibernate.cfg.xml<br /> getSessionFactory()<br />   {<br />    return sessionFactory;<br />   }<br />  }<br />  </pre><!-- =============main class for running the example ================= -->  <br /><h4>Run the
