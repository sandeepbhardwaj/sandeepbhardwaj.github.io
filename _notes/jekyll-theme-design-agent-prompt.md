Your task is to design the next iteration of an existing custom Jekyll theme for my technical blog, not to replace the platform or introduce a new framework.

Context:
- The site has already been migrated off Minimal Mistakes to local layouts, includes, SCSS, and vanilla JS.
- The theme must remain compatible with the current content model, front matter, integrations, and GitHub Pages-oriented Jekyll workflow.
- The blog focuses on backend engineering, system design, DSA, Java, and long-form developer education.

Non-negotiables:
- Do not break existing permalinks or page structures.
- Do not require mass edits to existing posts.
- Preserve search, TOC, Mermaid, code copy, comments, analytics, AdSense behavior, SEO, archives, pagination, and theme toggle.
- Use plain SCSS/CSS and vanilla JS unless there is a compelling reason not to.

What to produce:
1. Recommended visual direction for the evolved theme
2. Design system tokens and editorial style rules
3. Improved page wireframes for home, post, archive, about, search
4. Component contracts for header, post hero, TOC, code block, tables, callouts, taxonomy, related posts, footer
5. Content ergonomics guidance for code-heavy posts, summary tables, admonitions, and diagrams
6. Mobile and accessibility behavior
7. Performance/SEO guidance
8. A phased implementation plan that maps cleanly onto Jekyll layouts/includes/SCSS files
9. Migration-safe recommendations: what to refine first, what to defer, what not to touch
