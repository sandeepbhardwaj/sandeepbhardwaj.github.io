# Bytecode Engineering with ASM — Practical Intro

ASM enables low-level bytecode generation/transformation for frameworks and tooling.
Use it where source-level mechanisms are insufficient.

---

## Practical Use Cases

- runtime proxies without reflection cost
- instrumentation or telemetry weaving
- compile/build-time bytecode enhancement

---

## Basic ASM Flow

```java
ClassWriter cw = new ClassWriter(ClassWriter.COMPUTE_FRAMES);
cw.visit(Opcodes.V17, Opcodes.ACC_PUBLIC, "Hello", null, "java/lang/Object", null);
byte[] bytecode = cw.toByteArray();
```

---

## Engineering Rules

- keep generated bytecode minimal and auditable.
- validate stack frames and class verification early.
- pin ASM versions to JDK compatibility matrix.
- maintain golden tests for generated class behavior.

---

## Key Takeaways

- ASM is a precision tool for framework-level engineering.
- correctness and compatibility tests are mandatory.
- keep bytecode transformations narrow and explicit.
